# Contributing

This document records repository contribution conventions.

## Commit pattern

The commit description must be objective and clear. Very long messages are not recommended.

- `[Feat] New feature added`
- `[Doc] Documentation update`
- `[Fix] Bug Fix`
- `[Refactor] Code refactor, no new feature`
- `[Revert] Rollback to previous Commit`
- `[Test] Test has been included`

---

## Branches

- The default branch is `main`. 
- Sync the branch with `main` before opening or merging a pull request when needed.
- Direct development on `main` should be avoided. 
- Prefer short-lived branches over long-running branches.

### Branch Naming 

- `feat/new-awsome-feature`
- `fix/terrible-bug-discovered`
- `docs/new-imporant-documentation`
- `test/ci-cd-specific`
- `refactor/soma-ineficient-code-block`
- `chore/library-update`
